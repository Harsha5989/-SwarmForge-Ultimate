# Submission Checklist

Based on the submission requirements, ensure you have all these deliverables ready.

## 1. Project Deck (PDF)
- [ ] Format: PDF only
- [ ] Max 10 slides
- [ ] File Size Limit: 20 MB
- [ ] File Name: `TeamName_Deck.pdf`
- [ ] **Must Include:**
  - Problem statement
  - Solution overview
  - Architecture diagram (you can use the ASCII diagram from README or draw a nice one)
  - AI integration details (explain the LiteLLM proxy, Groq + OpenRouter multi-agent swarm)
  - Demo screenshots (take screenshots of the UI and the generated calculator)
  - Team introduction

## 2. Demo Video
- [ ] Format: MP4
- [ ] Max length: 3 minutes
- [ ] Resolution: Minimum 720p
- [ ] Delivery: Upload to YouTube (unlisted) OR provide a direct link
- [ ] **Must Include:**
  - Live walkthrough of the working prototype (NOT a slideshow)
  - Voiceover explaining the solution

## 3. GitHub Repository
- [ ] Visibility: Public repository
- [ ] Contains a comprehensive `README.md` (We have updated this for you!)
- [ ] **README.md Must Include:**
  - [x] Project description (Present in README)
  - [x] Setup instructions (Present in README)
  - [x] Dependencies (Present in README)
  - [x] Architecture overview (Present in README)
  - [x] AI tools used (Present in README)
  - [ ] Team member details with roles (Placeholder added in README, **make sure to fill it out!**)

## 4. Prototype / Live Link
- [ ] Working URL or deployment link where judges can interact with your solution
- [ ] Must be accessible for at least 30 days after the submission deadline
- [ ] (If you don't have a live cloud deployment, consider deploying the backend via Render/Railway and the frontend via Vercel/Netlify, or just provide ngrok links if necessary, though a real deployment is better.)
